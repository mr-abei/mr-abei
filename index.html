查看MySQL安装位置

DOM窗口登录mysql，输入：

show variables like “%char%”;

就可以看到了：

服务器 - 连接设置

服务器的数据库为了安全起见，一般不允许root账户远程登录，并设置不同权限（或只读或可修改）的子账户加以设置安全性高的密码作为远程登录账户.

创建只读权限用户用于远程登录

主要用到“grant”（授权）

MySQL [mysql]> use mysql;

  • 创建一个admin用户，密码是123456（当然，密码要复杂点，这里只是演示用，所以简单点）:

MySQL [mysql]> create user admin IDENTIFIED by ‘123456’;

  • 将数据库名为testdb的select权限给admin用户（admin是用来远程登录的，只给查询条件，不给update，delete等权限，这样不怕被恶意修改数据库）:

MySQL [mysql]> grant select on testdb.* to admin;

禁止root远程登录

MySQL [mysql]> delete from user where user=”root” and host!=”localhost”;

上面都设置好了之后，记得:

刷新数据库权限

flush privileges;

这一步无论是改密码还是授权用户都需要。

这样项目里就可以用root用户连接，拥有所有权限，而平时开发的时候可以用admin用户查看数据库。

好了，做到这里发现炸了，上面那个划掉的语句没输好，导致把整个root用户误删掉了。现在只有一个刚创建的只读可远程连接账户，然并卵，操作数据表都没得。
所以进入下面主题

误删Mysql中误删管理员root账户的解决之道

现在已经没办法对数据库进行操作了，所以需要一个有权限的账户对数据库进行操作。

所以进行以下步骤：

回车

2. 创建 root 用户并赋权限，并修改root密码

执行完上面三步就ojbk啦

3. 刷新权限

4.授权
5.查看用户权限

select * from mysql.user\G

或者

select user,password,host from user;

5.7版本的话是没有password的，换成了authentication_string，替上即可。

/g 表示竖着排列展示

6.创建用户
6.1. 添加用户

跟以往版本不同，MySQL5.7 mysql.user表没有password字段，这个字段改成了 authentication_string；
这里我们使用命令进行创建用户：

CREATE USER ‘username’@’host’ IDENTIFIED BY ‘password’;

如创建一个test用户，密码为test123，可以进行远程登录：

  • username - 你将创建的用户名
  • host - 指定该用户在哪个主机上可以登陆，此处的”localhost”，是指该用户只能在本地登录，不能在另外一台机器上远程登录，如果想远程登录的话，将”localhost”改为”%”，表示在任何一台电脑上都可以登录;也可以指定某台机器可以远程登录;
  • password - 该用户的登陆密码,密码可以为空,如果为空则该用户可以不需要密码登陆服务器。
    6.2. 删除用户

如果用户创建错了，肯定要支持删除操作，使用命令：

DROP USER ‘username’@’host’;

6.3. 授权

以下按例子说明：

授权test用户有testDB数据库的某一部分权限：

grant select,update on testDB.* to test@’%’ identified by ‘test123’;

授权test用户有testDB数据库的所有操作权限：

grant all privileges on testDB.* to ‘test’@’%’ identified by ‘test123’;

授权test用户拥有所有数据库的某些权限：

  • privileges - 用户的操作权限,如select,delete,update,create,drop等(详细列表可自行百度)，如果要授予所有的权限可使用all（参考第二种授权方式）;

  • % 表示对所有非本地主机授权，不包括localhost。

Navicat(MySql)错误1045 Access denied for user ‘root’@’localhost’ (using password:YES)

放完周末第一天上班，打开电脑听前端小姐姐说网站挂了，远程连接进入服务器（win系统），发现服务器莫名重启了，然后项目都没在运行，这些不是事儿，启动即可。重点是Navicat连接数据库连不上了，命令行窗口是可以的，略显棘手。开始分析原因：

1. 密码错误

命令行能进去，说明密码没错，重输入几次也一样更确定密码没敲错。

2. 端口没开放

去高级安全 windows防火墙–> 入站规则–> 新建规则–> 规则类型–> 端口 –> TCP、特定本地端口 输入3306–> 允许连接 –> …保存。

排除这个原因，继续分析。

3. 端口错误

首先想到的是端口号是不是被占用了。解决办法：确认Navicat填的端口号和MySQL服务端口号一致

- 使用命令进行登录

mysql -uroot -p

- 查看mysql使用的端口号

show global variables like ‘port’;


端口号还是3306没错，继续分析。

4. 权限问题

想到用户权限的问题，是否没有访问权限、或者远程连接权限。
命令行登录mysql，use mysql

- 查看连接权限

select user,authentication_string,host from user;

看到root是host是“%”，也就是开放了远程连接。还有开出来的账户也是有权限。

顺便

- 查看MYSQL数据库中所有用户

mysql> SELECT DISTINCT CONCAT(‘User: ‘’’,user,‘’’@’’’,host,’’’;’) AS query FROM mysql.user;

- 查看数据库中具体某个用户的权限

mysql> show grants for ‘用户名‘@’%’;

- 查看一下操作权限

select * from mysql.user where user=’用户名’ \G

没问题，继续分析。

5. 服务没启动

打开任务管理器，服务是有启动的。
